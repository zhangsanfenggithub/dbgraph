DbGraph Usage Guide

DbGraph is a local-first database context tool. The normal workflow is: initialize a project, configure a provider, capture a snapshot, then use the local graph for search, validation, impact analysis, and review reports.

For all .dbgraph/dbgraph.config.json fields, see the configuration reference.

Install or run

# macOS / Linux
curl -fsSL https://raw.githubusercontent.com/zhangsanfenggithub/dbgraph/master/install.sh | sh
# Windows PowerShell
irm https://raw.githubusercontent.com/zhangsanfenggithub/dbgraph/master/install.ps1 | iex

Initialize a project

dbgraph init -i --yes
dbgraph install --target codex --yes
dbgraph status

This creates .dbgraph/dbgraph.config.json, .dbgraph/snapshots/, .dbgraph/instructions/, and .dbgraph/dbgraph.db after the first successful snapshot.

Configure a database

Project config has four top-level sections: database, snapshot, security, and mcp.

PostgreSQL

$env:DATABASE_URL="postgres://postgres:postgres@localhost:55432/teashop"
dbgraph snapshot --profile stats

SQLite

SQLite does not require an external service. Set provider to sqlite and connectionString to your business database file in .dbgraph/dbgraph.config.json.

SQL artifacts

During snapshot, DbGraph scans migrations/, sql/, and db/ for SQL files. SQL artifacts become query objects in the graph, with read, write, filter, and join dependencies where supported.

Daily CLI commands

dbgraph search customer
dbgraph table public.orders
dbgraph relations public.orders --depth 2
dbgraph context "refund payment order" --tokens 800
dbgraph validate-sql --sql "select * from orders"
dbgraph diff
dbgraph impact public.orders.status

Analysis reports

dbgraph analyze --scope all
dbgraph analyze --scope risk
dbgraph analyze --scope all --format markdown --output dbgraph-analysis.md

Reports include overview, risk score, section summaries, top findings, severity counts, evidence, impact, confidence, suggested fixes, and related SQL/schema objects.

MCP server

dbgraph serve --mcp

MCP tools include dbgraph_status, dbgraph_search, dbgraph_table, dbgraph_context, dbgraph_relations, dbgraph_impact, dbgraph_analyze, dbgraph_diff, and dbgraph_validate_sql.

DbGraph 使用说明

DbGraph 是一个本地优先的数据库上下文工具。典型流程是初始化项目、配置数据库 provider、生成 snapshot，然后基于本地图索引执行搜索、SQL 校验、影响分析和审查报告。 完整 .dbgraph/dbgraph.config.json 字段说明见 Configuration。

安装或直接运行

推荐方式不依赖 Node.js，也不需要本机安装 Rust toolchain。

# macOS / Linux
curl -fsSL https://raw.githubusercontent.com/zhangsanfenggithub/dbgraph/master/install.sh | sh
# Windows PowerShell
irm https://raw.githubusercontent.com/zhangsanfenggithub/dbgraph/master/install.ps1 | iex

初始化项目

dbgraph init -i --yes
dbgraph install --target codex --yes
dbgraph status

Agent MCP 配置会写入 command: "dbgraph"，所以运行 dbgraph install 前需要先让 dbgraph 出现在 PATH。

配置数据库

PostgreSQL 默认使用 DATABASE_URL：

$env:DATABASE_URL="postgres://postgres:postgres@localhost:55432/teashop"
dbgraph snapshot --profile stats

SQLite 不需要外部服务。把 .dbgraph/dbgraph.config.json 中的 provider 设置为 sqlite，并填写业务数据库文件路径即可。

常用命令

dbgraph search customer
dbgraph table public.orders
dbgraph relations public.orders --depth 2
dbgraph context "refund payment order" --tokens 800
dbgraph validate-sql --sql "select * from orders"
dbgraph diff
dbgraph impact public.orders.status
dbgraph analyze --scope all --format markdown --output dbgraph-analysis.md

安全说明

  • dbgraph validate-sql 不会执行 SQL。
  • dbgraph analyze 只基于本地 snapshot 和图索引工作。
  • dbgraph snapshot 是会连接配置数据库的命令。
  • 默认关闭 raw sample 存储。
  • 显式开启采样时，敏感样本默认会被 mask。

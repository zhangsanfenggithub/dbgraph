Security Model

DbGraph is designed as a local, read-only database context tool.

Boundaries

  • dbgraph validate-sql never executes SQL.
  • dbgraph analyze works from the local snapshot and graph index.
  • dbgraph snapshot is the command that connects to the configured database.
  • Raw sample storage is off by default.
  • Sensitive samples are masked when sampling is explicitly enabled.
  • Sensitive connection strings are not printed in provider errors.
  • SQLite provider refuses to snapshot .dbgraph/dbgraph.db.

PII analysis

DbGraph can flag sensitive-looking columns and SQL artifacts that reference those columns. Findings include evidence, confidence, related objects, and suggested fixes.
